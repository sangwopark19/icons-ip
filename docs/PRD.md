## 1. 개요

**ICONS**는 서브컬처 팬덤을 위한 **슈퍼앱**이다. 공식 라이선스 굿즈 커머스를 척추로, 수집형 디지털 카드(가챠), 팝업/티케팅, 팬 커뮤니티, IP 허브를 하나의 "Holographic Midnight" 경험으로 묶는다.

현재 리포(`/Users/sangwopark19/icons/icons-ip`)는 Claude Design 핸드오프를 옮긴 **시각적 프로토타입**에서 출발했다. 11개 화면은 App Router 라우트로 구현되어 있고, Supabase 공개 카탈로그 읽기, 이메일/PW Auth, 확인 메일 콜백, 온보딩, IP 팔로우 일부가 실제 backend에 연결되어 있다. 결제·주문 확정·가챠 RNG·티케팅 검표는 아직 실제 서비스에 연결되지 않았다. 이 PRD는 프로토타입을 **실제 출시 제품**으로 전환하기 위한 요구사항을 정의한다.

### 1.1 제품 비전

> "흩어진 서브컬처 팬덤의 소비·수집·소통을 한 곳에서 — 공식 라이선스의 신뢰 위에서."

### 1.2 목표 (v1)

- 공식 라이선스 굿즈를 안전하게 **구매**할 수 있다 (장바구니 → 토스페이먼츠 결제 → 주문).
- IP별 **수집형 디지털 카드**를 유료 가챠로 뽑고 **바인더**에 모은다.
- 팝업/팬미팅 **티켓을 예매**하고 전자티켓(QR)으로 입장한다.
- 관심 IP를 **팔로우**하고 팬들과 **커뮤니티**에서 소통한다.

### 1.3 비목표 (v1에서 하지 않는 것)

- **C2C 마켓**(중고 거래)·**교환**(직거래·경매): UI는 유지하되 거래·정산·에스크로는 구현하지 않는다 → **v2**.
- **유료 멤버십/구독**(IP 팬덤 유료 티어) → v2.
- 실시간 채팅/라이브 피드 → 검토 후 차기.
- 글로벌(다국어·해외결제) 본격 대응 → 차기.

## 2. 대상 사용자

| 페르소나 | 설명 | 핵심 니즈 |
|---|---|---|
| **수집러** | 포토카드·홀로 카드 컬렉션에 진심인 팬 | 가챠의 짜릿함, 보유 현황(바인더), 한정 드롭 알림 |
| **굿즈 구매자** | 공식 라이선스 굿즈를 정품으로 사고 싶은 팬 | 신뢰(정품)·재고·빠른 결제 |
| **이벤트 참여자** | 팝업·팬미팅에 가는 팬 | 회차/잔여 확인, 간편 예매, 전자티켓 |
| **커뮤니티 유저** | 인증샷·후기·교환글로 소통하는 팬 | 작성·반응·신고/차단 안전장치 |
| **운영자(내부)** | IP·상품·카드풀·이벤트·신고를 관리 | 백오피스(`/admin`)에서 카탈로그·확률 공시·모더레이션 |

브라우징은 **로그인 없이 공개**. 액션(구매·가챠·예매·작성·팔로우) 시점에 로그인을 요구한다.

| 팝업 | 이벤트 목록 UI만 | 실제 티케팅(예약·결제·QR·환불) |
| 커뮤니티 | 공개 피드와 IP 상세 preview 읽기, 로그인·온보딩 완료 사용자 포스트 작성(선택 이미지 업로드 포함)이 Supabase에 연결. posts/comments/likes/reports/blocks 스키마는 있음 | 댓글·좋아요·신고/차단 UI와 액션 |
| 아트워크 | 그라디언트+글리프 플레이스홀더 | 실제 이미지(Supabase Storage) |

## 4. 범위

### 4.1 v1 포함

1. 인증 & 온보딩
2. IP 허브 (브라우징 + 무료 팔로우)
3. 굿즈 커머스 (장바구니 · 결제 · 주문) — **척추**
4. 가챠 & 수집형 카드 (디지털 유료) + 바인더
5. 팝업/티케팅 (실제 예매 · 전자티켓)
6. 커뮤니티 (작성형)
7. 검색
8. 운영 백오피스(`/admin`)

### 4.2 v2 이후

- C2C 마켓 · 교환(직거래·경매) — 에스크로·정산·분쟁·통신판매중개
- 유료 멤버십/구독
- 실시간 기능, 글로벌 확장, PASS 본인인증(필요 시 v1.x로 당김)

## 5. 기능 요구사항

표기: **P0/P1/P2/P3**는 [§9 출시 단계](#9-출시-단계)의 빌드 순서. `M`=필수(MUST), `S`=권장(SHOULD).

### 5.1 인증 & 온보딩 (P0)

- `M` 이메일/비밀번호 가입·로그인.
- `M` 이메일 확인 메일 콜백과 재전송. 같은 브라우저에서 같은 이메일로 반복 가입을 시도하면 최신 확인 메일을 재발송하되, 과도한 반복은 10분 window로 제한한다.
- `M` 소셜 로그인: **Google · Apple · Kakao**. 현재 구현은 이메일/PW까지이며 소셜 버튼은 비활성 상태다.
- `M` 소셜 로그인이어도 **온보딩에서 이메일·닉네임·생년월일(자가신고)·약관/마케팅 동의를 수집**하여 프로필을 완성한다.
- `M` 공개 페이지는 비로그인 접근 가능. 보호 액션은 로그인 게이트.
- `M` 본인확인은 **자가신고 + 결제 시 결제사(토스페이먼츠) 위임**. 별도 PASS 연동 없이 시작(게임물 등급에서 연령제한 요구 시 v1.x에 PASS 추가).
- `S` 로그아웃·계정 삭제·동의 철회.

### 5.2 IP 허브 (P0)

- `M` IP 목록/상세를 **비로그인 공개** 브라우징.
- `M` IP 상세: 배너·태그라인·시놉시스·지표(팬/굿즈/카드/이벤트), 탭(굿즈·카드·팝업·커뮤니티).
- `M` **무료 팔로우**(관심 IP 등록) → 홈/피드/알림 개인화.
- `S` 알림 설정(IP별 드롭·이벤트 알림 on/off).
- 버티컬: BL/GL · 로맨스판타지 · 글로벌 IP · 버튜버 · 스트리머 (버티컬별 네온 액센트 유지).

### 5.3 굿즈 커머스 (P1) — 척추

- `M` 굿즈 목록(타입·IP·정렬·재고 상태), 굿즈 상세.
- `M` 장바구니: 담기/수량/삭제, 비로그인 시 로컬 → 로그인 시 병합.
- `M` 체크아웃: 배송지·결제수단 선택 → **토스페이먼츠 결제** → 주문 생성.
- `M` 재고 차감은 **결제 확정 시점에 원자적으로** 처리(품절 동시성).
- `M` 주문 내역·상태(결제완료/배송중/완료/취소).
- `M` **청약철회/환불**(전자상거래법) 처리 흐름.
- `S` 한정·신상·예약·품절 배지, 한정 수량 표시.

### 5.4 가챠 & 수집형 카드 (P2)

- `M` **지갑**: 토스페이먼츠로 캐시/포인트 **충전**, 잔액·거래내역.
- `M` **가챠 뽑기**: 카드풀에서 RNG 추첨, **단챠/연챠**, 결과 연출.
- `M` **천장(pity)**: 일정 횟수 보장(예: N회 내 최고 등급 1장). 천장 카운터 노출.
- `M` **확률 공시**: 각 카드풀의 등급별 확률을 화면에 명시(게임산업진흥법 확률형 아이템 공시 의무).
- `M` **바인더**: 보유/미보유 카드, 등급(N·R·SR·SSR·HOLO), 도감 진행률, IP/등급 필터.
- `M` 미사용 충전금 **환불 정책**(전자금융/약관) 노출 및 처리.
- `S` 듀플리킷(중복) 처리 규칙(포인트 환원 등), 픽업/한정 풀.
- 규제 의존: [§7](#7-규제--법무-요구사항) 참조. 게임물 등급분류 결과가 본 도메인의 노출 조건(연령)을 좌우.

### 5.5 팝업/티케팅 (P3)

- `M` 이벤트 목록/상세(모드: 온/오프라인, 상태: 예매중/예정/진행중, 일시·장소).
- `M` **티켓 회차/종류·수량** 정의(운영자), **잔여 수량** 표시.
- `M` 예매: 수량 선택 → **재고 원자적 차감** → 토스페이먼츠 결제 → **전자티켓(QR) 발급**.
- `M` 입장: QR **검표**(중복 사용 방지), 사용/미사용 상태.
- `M` **예매 취소·환불** 정책(일정·수수료 규칙).
- `S` 1인 구매 한도, 예매 오픈 시각, 대기열(고동시성 회차).

### 5.6 커뮤니티 (P0)

- `M` 글 **작성**(텍스트 + 이미지 업로드), 태그, IP 연결.
- `M` 댓글, 좋아요.
- `M` **신고 · 숨김 · 차단** (UGC 안전장치).
- `M` 운영자 모더레이션(`/admin`에서 신고 처리·게시물 비공개).
- `S` 트렌딩 태그, 팔로우 IP 기반 피드 개인화.
- 정책: 명예훼손·불법촬영물·저작권 침해 신고 대응 절차(운영 정책 문서로 분리).

### 5.7 검색 (P0~)

- `M` 통합 검색: IP·굿즈·카드·포스트·태그.
- `M` 구현은 **Postgres pg_trgm/ILIKE**(한국어 부분일치). 결과 그룹핑.
- `S` 최근 검색어·인기 검색어.

### 5.8 운영 백오피스 `/admin` (각 도메인과 병행)

- `M` 역할 기반 접근(`admin`/`staff`). 일반 사용자 차단.
- `M` 카탈로그 관리: IP·굿즈·카드·카드풀(+확률 공시값)·이벤트·티켓 회차.
- `M` 주문/티켓/환불 조회 및 처리.
- `M` 커뮤니티 신고 처리.
- `M` **감사 로그**(audit_log): 민감 작업 기록.

## 6. 핵심 사용자 플로우

1. **첫 방문(비로그인)**: 홈 → IP 탐색 → 굿즈/카드 구경(공개). 액션 시 로그인 유도.
2. **온보딩**: 소셜/이메일 로그인 → 이메일·닉네임·생년월일·동의 입력 → 관심 IP 팔로우.
3. **굿즈 구매**: 굿즈 상세 → 장바구니 → 체크아웃 → 토스 결제 → 주문완료 → 내역.
4. **가챠**: 지갑 충전(토스) → 카드풀 진입(확률 공시 확인) → 단챠/연챠 → 결과/천장 → 바인더 적립.
5. **티켓**: 이벤트 상세 → 회차/수량 선택 → 토스 결제 → 전자티켓(QR) → 현장 검표.
6. **커뮤니티**: 피드 → 글 작성(이미지) → 반응/댓글 → (문제 시) 신고/차단.

## 7. 규제 · 법무 요구사항

선택한 기능(유료 디지털 가챠 + 결제 + 선불 충전 + UGC)은 다음 의무를 동반한다. 제품·법무가 출시 전 확인해야 하는 **요구사항**으로 본다.

| 항목 | 적용 | 요구사항 |
|---|---|---|
| **전자상거래법** | 굿즈·티켓·가챠 | 사업자/판매자 정보·약관·청약철회·표시광고 준수, **통신판매업 신고** |
| **확률형 아이템 공시** (게임산업진흥법) | 디지털 유료 가챠 | 카드풀 등급별 **확률 공시** 화면 노출 의무 |
| **게임물 등급분류** (게임물관리위원회) | 디지털 가챠 | 가챠가 "게임물"로 분류될 수 있음 → **등급분류 필요 여부 검토**(출시 블로커 가능성). 연령제한 시 §5.1에 PASS 본인인증 추가 |
| **전자금융거래법** | 선불 충전(지갑) | 선불전자지급수단 해당 여부·한도, **미사용 충전금 환불** 정책 |
| **개인정보보호법(PIPA)** | 전체 | 수집 항목 최소화·동의·보관/파기, 만 14세 미만 처리 정책 |
| **청소년보호법** | 가챠·결제 | 연령 확인(자가신고+결제사), 결제 한도, 등급분류 결과 연동 |
| **저작권/라이선스** | IP·굿즈·아트워크 | 공식 라이선스 범위 내 사용, 침해 신고 대응 |

> **결정 기록 후보**: "디지털 유료 가챠 채택 + 규제 스탠스"는 되돌리기 어렵고 트레이드오프가 명확하여 별도 ADR로 남길 가치가 있다(§11).

## 8. 비기능 요구사항

- **무결성**: 가챠 뽑기·티켓 재고·주문·지갑은 동시성에서도 **원자적**이어야 한다(이중 차감·초과 판매 금지). → Postgres RPC + 행 잠금([아키텍처](./ARCHITECTURE.md) 참조).
- **결제 신뢰성**: 결제 확정은 **웹훅 기반 + 멱등 처리**. 클라이언트 성공 신호만으로 주문을 확정하지 않는다.
- **보안**: 사용자 데이터는 Supabase **RLS**로 소유자 범위 격리. 카탈로그는 공개 읽기. 관리자 작업은 역할+감사 로그.
- **성능**: 공개 카탈로그는 캐시 가능(SSR/정적). LCP 목표 양호 수준, 이미지 최적화.
- **가용성/관측성**: 결제·가챠·티켓 트랜잭션 로깅, 실패 알림.
- **접근성/반응형**: 모바일 우선(MobNav), 데스크톱 Nav. 한국어 기본.
- **디자인 일관성**: "Holographic Midnight" 디자인 시스템(`app/globals.css`) 유지.

## 9. 출시 단계

범위는 v1로 동일하되, **위험도·의존성 순**으로 단계 출시한다.

| 단계 | 내용 | 트랜잭션 |
|---|---|---|
| **P0 — 기반** | Supabase Auth·DB·RLS, 카탈로그 브라우징(IP/굿즈/카드/이벤트), IP 팔로우, 커뮤니티(작성형), 검색, `/admin` 골격 | 비결제 |
| **P1 — 커머스** | 장바구니·체크아웃·**토스페이먼츠**·주문·환불, 재고 차감 | 결제 #1 |
| **P2 — 가챠/지갑** | 지갑 충전, RNG·천장, **확률 공시**, 바인더 적립, 충전금 환불 | 결제 #2 (선불) |
| **P3 — 티케팅** | 회차/재고, 예매·결제, 전자티켓(QR)·검표, 예매 환불 | 결제 #3 |

각 단계는 독립 출시 가능하도록 설계한다. 규제 검토(가챠 등급분류)는 P2 출시의 선행 조건.

## 10. 성공 지표 (초안)

- 활성화: 가입→온보딩 완료율, 관심 IP 1개 이상 팔로우 비율.
- 커머스: 장바구니→결제 전환율, 결제 성공률.
- 가챠: 충전 전환율, 1인당 뽑기 수, 바인더 도감 진행률.
- 티켓: 예매 성공률, 검표 정상률.
- 커뮤니티: 작성/반응 비율, 신고 처리 시간.
